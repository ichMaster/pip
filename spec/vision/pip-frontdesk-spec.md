# pip — фронт-деск для агентів

Розвиток pip із чат-тамагочі в живий термінальний інтерфейс до флоту агентів: ти питаєш pip, а вона довідується, запускає й моніторить агентів — лишаючись істотою з потребами і голосом.

> База незмінна: pip — істота в терміналі (тіло + потреби + LLM як голос), принцип двох годинників — тіло тікає само, розум запитуємо лише коли є що сказати. Тепер розум уміє ще й маршрутизувати до агентів.

---

## 1. Що змінюється

- pip перестає бути однією істотою і стає **фронт-деском**: каталог агентів, концьєрж-розум, адаптери до агентів, спільний журнал флоту.
- forage (потребо-керований ресерч-агент із попереднього спеку) перепозиціонується: це не окремий пакет, а **перший зареєстрований у pip робочий агент**. pip — його фронт-деск.
- Тамагочі-душа лягає на флот: потреби pip починають дзеркалити стан агентів (простій підіймає неспокій, успіхи підіймають настрій).

---

## 2. Три наміри концьєржа

Коли ти пишеш pip, її LLM-розум розрізняє намір (із rule-фолбеком без API):

| Намір | Що робить | Джерело |
|---|---|---|
| Розмова | відповідає в характері, як зараз | персона pip |
| Довідка | відповідає про агентів: хто є, що вміє, що робить, що знайшов | реєстр + статус + журнал |
| Диспатч | дає агенту задачу, підтверджує, запускає, приносить результат | адаптери |
| Турбота | `/feed` `/play` `/sleep` `/status` — як зараз | ядро pip |

Вивід розуму = структурована дія (який агент, яка ціль) + репліка голосом у характері.

## 4. Реєстр агентів (довідник)

Маніфест `agents.toml` — серце «довідкової» частини. pip знає каталог.

```toml
[[agent]]
id = "forage"
name = "forage"
blurb = "нишпорить за наукою, ведений цікавістю"
tags = ["research", "science", "web", "synthesis"]
adapter = "daemon"          # subprocess | sdk | daemon
endpoint = "~/.pip/agents/forage.sock"
status_source = "journal"   # звідки брати стан

[[agent]]
id = "mote"
name = "mote"
blurb = "істота живе у власному ASCII-світі"
tags = ["alife", "sim"]
adapter = "subprocess"
cmd = "python -m mote.run_headless"
```

- `pip agents` — список з blurb і статусом.
- Запит «хто вміє X» -> матч за `tags`.

## 5. Контракт адаптера

Будь-який агент підключається через єдиний інтерфейс:

```python
class AgentAdapter(Protocol):
    def status(self) -> AgentStatus: ...          # живий? зайнятий? потреби/енергія?
    def dispatch(self, goal: str) -> Handle: ...   # дати задачу
    def result(self, handle: Handle) -> DispatchResult | None: ...

@dataclass
class AgentStatus:
    alive: bool
    busy: bool
    summary: str           # людський рядок для довідки pip
    extra: dict            # напр. forage: needs, energy

@dataclass
class DispatchResult:
    text: str
    artifacts: list[Path]
    ok: bool
```

Бекенди:
- `subprocess` — `claude -p` або разовий процес (mote).
- `sdk` — `claude-agent-sdk` для контролю.
- `daemon` — довгоживучий агент (як forage), pip питає його через сокет/файл.

---

## 6. Журнал флоту

Спільне місце, звідки pip читає, щоб відповідати на довідку і підсумовувати.

- `~/.pip/fleet/status.json` — поточні статуси агентів (кожен агент пише свій).
- `~/.pip/fleet/journal/` — знахідки/результати (markdown), читаються для «що зробив X».
- pip не дублює дані агентів, лише читає й переказує голосом.

---

## 7. Потреби pip, прив'язані до флоту

Базові потреби лишаються; додаються зв'язки з флотом (тримати мінімально):

| Потреба | Базова динаміка | Зв'язок із флотом |
|---|---|---|
| Настрій (`mood`) | дрейф до нейтралі, реакція на тебе | успіх агента -> вгору; падіння/помилка -> вниз |
| Енергія (`energy`) | regen у часі | спадає, коли pip спавнить/опитує агентів |
| Неспокій (`restlessness`) | повільний ріст | різко росте, коли агенти довго простоюють |

Коли `restlessness` високий, pip сама подає голос: «forage давно без теми, спустити його?». Це зберігає тамагочі-поведінку, але робить її осмисленою щодо роботи.

## 9. Структура пакета

```
pip-project/
  pyproject.toml
  src/pippet/                 # import-ім'я НЕ "pip" (див. застереження)
    core/   body.py needs.py mood.py state.py
    brain/  concierge.py intents.py persona.py
    registry/ catalog.py        # читає agents.toml
    adapters/ base.py subprocess.py sdk.py daemon.py
    fleet/  journal.py status.py
    tui/    face.py panel.py fleet_panel.py
    cli.py                      # pip run | agents | ask | dispatch | feed
  agents.toml
```

> Застереження: пакет не можна назвати `pip` — це конфлікт з менеджером пакетів. Distribution-ім'я лишай «pip» (або `pip-pet`), import-ім'я — `pippet`. Консольний скрипт `pip` теж зайнятий; реєструй entry point як `pippet` (або `pet`).

## 11. Відкриті рішення

- Персона: pip-істота й pip-концьєрж — один голос чи два регістри (грайливий у розмові, діловіший у довідці)?
- Матч агента: rule по tags для старту; LLM-матч додати, коли агентів багато.
- Стан forage: тягнути pull-ом при запиті чи forage пушить у `status.json` періодично (дешевше для pip).
- Межа турботи vs керування: чи `/feed` тепер годує енергію pip на спавн агентів (бюджет = їжа), як у forage.
