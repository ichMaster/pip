> Source of truth: the **v0.1** section of [spec/ROADMAP.md](../../ROADMAP.md).
> This file is the fine-grained breakdown consumed by `/upload-issues`.

**Goal:** the face + needs become one **persistent picture that updates in place**
(like Claude Code's stable UI), not a fresh face block printed after every turn.

Today [`run()` / `_handle`](../../../creature/chat.py) re-`print(status_line(…))` on
every interaction ([chat.py:42](../../../creature/chat.py#L42),
[chat.py:56](../../../creature/chat.py#L56),
[chat.py:91](../../../creature/chat.py#L91),
[chat.py:94](../../../creature/chat.py#L94)), so the terminal fills with a stack of
stale faces.

**Scope & guardrails (from the roadmap and [CLAUDE.md](../../../CLAUDE.md)):**
- This is **loop / presentation work in [`chat.py`](../../../creature/chat.py)**. The
  pure render functions in [`render.py`](../../../creature/render.py) (`face_state`,
  `face_block`, `bar`, `status_line`) **stay unchanged**.
- [`render.py`](../../../creature/render.py) remains the **sole owner of ANSI / box-drawing** —
  any new cursor-control escape sequences live there, not in `chat.py`.
- **Two-clock split intact:** the body ([`needs.py`](../../../creature/needs.py)) is not
  touched; this changes only *how* the body's state is drawn.
- **Stdlib only.** No new dependency. **Degrade gracefully** where ANSI is ignored or
  output is not a TTY (e.g. piped input) — fall back to an occasional reprint.
- **Forward link:** precursor to `v1.1`'s live-client full-frame redraw. Keep the
  redraw logic simple and contained; the full sticky-frame/cbreak treatment is v1.1.
- **Release:** completing this phase cuts `0.1.0` (via `/release-version`, user's call).

## PIP-001: Cursor-control ANSI primitives in render.py

### Description
Add the small set of ANSI escape-sequence helpers needed to redraw a fixed region of
the terminal in place. [`render.py`](../../../creature/render.py) is the project's
**sole owner of ANSI / box-drawing**, so these primitives belong here — `chat.py` will
*use* them but never emit raw escape codes itself.

### What needs to be done
- In [`render.py`](../../../creature/render.py), add pure, string-returning helpers next
  to the existing `_ansi` / color definitions:
  - `cursor_up(n)` — move the cursor up `n` lines (`\033[{n}A`); returns `""` for `n <= 0`.
  - `clear_line()` — erase the current line (`\033[2K`) and return to column 0 (`\r`),
    or expose `CLEAR_LINE` as a constant.
  - `hide_cursor()` / `show_cursor()` (`\033[?25l` / `\033[?25h`) — optional, to avoid
    cursor flicker during a redraw; include only if used by PIP-002.
- Add a `STATUS_HEIGHT` constant (= **4**, the number of lines `face_block` /
  `status_line` produce) so callers know how far to move the cursor up. Keep it derived
  from / consistent with `face_block` (which returns 4 lines).
- **Do not change** the existing pure functions (`face_state`, `face`, `face_block`,
  `bar`, `status_line`) — only add new helpers/constants.
- Keep everything **pure**: these helpers return strings; they perform no I/O.

### Dependencies
None.

### Expected result
`render.py` exposes cursor-up / clear-line (and optionally hide/show-cursor) helpers plus
a `STATUS_HEIGHT` constant, ready for `chat.py` to compose an in-place redraw. No behavior
change yet on its own.

### Acceptance criteria
- [ ] `cursor_up(n)` returns the correct escape string and `""` for `n <= 0`.
- [ ] A clear-line helper/constant exists and erases + returns to column 0.
- [ ] `STATUS_HEIGHT == 4` and matches the number of lines `status_line` renders.
- [ ] Existing render functions are byte-for-byte unchanged in behavior.
- [ ] `python -m py_compile creature/render.py` passes; `python -c "import creature.render"` works.
- [ ] No new dependency; ANSI still degrades gracefully (plain strings).

## PIP-002: Redraw the status block in place (no stacking)

### Description
Replace the scattered per-interaction `print(status_line(…))` calls with a single
**status region that is redrawn in place**, so across a session there is exactly one
status picture instead of a growing stack of stale faces. This is the core of v0.1.

### What needs to be done
- In [`chat.py`](../../../creature/chat.py), introduce a small helper (e.g.
  `_draw_status(name, needs, *, redraw)`) that:
  - On the **first** draw, prints the status block once.
  - On a **redraw**, uses the PIP-001 primitives to move the cursor up `STATUS_HEIGHT`
    lines, clear those lines, and reprint the current `status_line(name, needs)` — so the
    face and bars update **in place** without a new block scrolling in.
- Make the status block the redrawn region that sits directly above the input prompt;
  **chat lines scroll as normal** (replies via `say_line`, spontaneous lines, the
  `you>` prompt). Only the status block is rewritten in place.
- Remove the redundant `print(status_line(…))` calls at
  [chat.py:42](../../../creature/chat.py#L42),
  [chat.py:56](../../../creature/chat.py#L56),
  [chat.py:91](../../../creature/chat.py#L91), and
  [chat.py:94](../../../creature/chat.py#L94); route all of them through the new helper so
  feeding / playing / sleeping / chatting and spontaneous nags **update the one picture**
  rather than emitting a new block.
- Keep `render.py` pure and unchanged; all cursor bookkeeping (how many lines to move,
  when to redraw) lives in `chat.py`.
- **Body untouched:** this is presentation only — `pet.tick`, `Needs`, and the brains
  are not modified.

### Dependencies
- PIP-001 (cursor-control primitives + `STATUS_HEIGHT`).

### Expected result
Feeding, playing, talking, and spontaneous nags update a single in-place status picture.
The terminal no longer accumulates a stack of old faces during a session.

### Acceptance criteria
- [ ] Across a session the face stays as **one updating picture** — no stack of old faces.
- [ ] `/feed`, `/play`, `/sleep`, and a chat turn each **update the picture in place**
      (bars/face change without a new block appearing).
- [ ] Chat replies, spontaneous lines, and the `you>` prompt still scroll normally.
- [ ] The four `print(status_line(…))` sites are gone / routed through the new helper.
- [ ] Two-clock split intact: no change to `needs.py`, `pet.py`, or the brains; render
      functions unchanged.
- [ ] Smoke-test passes: `printf '/status\nhello\n/feed\n/quit\n' | python pip.py --rule --speed 5`.
- [ ] `python -m py_compile creature/chat.py` passes.

## PIP-003: Live idle refresh + non-TTY graceful fallback

### Description
Make the in-place picture **animate during idle drain** (the bars visibly drift while you
sit and think), and ensure the redraw **degrades gracefully** when output is not an
interactive terminal (piped input, ANSI ignored) so the documented smoke-test still works.

### What needs to be done
- In the [`run()`](../../../creature/chat.py#L38) loop, refresh the status region on a
  cadence (each loop pass / `POLL_SECONDS` tick, or on change) using the PIP-002 helper,
  so drain is visible live **without spamming output** (redraw in place, don't reprint).
- Avoid disturbing in-progress typed input: only redraw when it is safe to do so given the
  `select`-based cooked-mode input (e.g. redraw between polls; keep the prompt intact).
  Keep this simple — the full sticky-frame/cbreak treatment is deferred to `v1.1`.
- Detect non-interactive output (e.g. `sys.stdout.isatty()` is false): when not a TTY,
  **fall back to an occasional reprint** instead of cursor-control redraws, so piped runs
  and the smoke-test produce sane, non-garbled output.
- Optionally use `hide_cursor`/`show_cursor` (PIP-001) around redraws to avoid flicker;
  always restore the cursor on exit (including `/quit`, EOF, and `Ctrl-C`).

### Dependencies
- PIP-002 (the in-place redraw mechanism).

### Expected result
While idle, the bars visibly drift in place (fast with `--speed`), the face changes as
needs cross thresholds, and piped / non-TTY runs degrade to plain reprints without escape
soup.

### Acceptance criteria
- [ ] During idle, bars visibly drift **in place** over time; clearly faster with `--speed`.
- [ ] The face updates in place when a need crosses a `face_state` threshold while idle.
- [ ] Redraw does not garble or duplicate the prompt / in-progress input.
- [ ] Non-TTY / piped mode degrades gracefully (no raw escape codes in captured output);
      the smoke-test `printf '/status\nhello\n/feed\n/quit\n' | python pip.py --rule --speed 5` stays clean.
- [ ] Cursor is restored on `/quit`, EOF, and `Ctrl-C` (no hidden cursor left behind).
- [ ] `python -m py_compile creature/chat.py` passes.
